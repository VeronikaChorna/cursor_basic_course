Main commands

  • git config --global user.name blabla - is used to set Git username;
  • git config --global user.email blabal@bla - is used to set Git email address;
  • git init - creates a new Git repository;
  • git status - is used to display the current status of the local repository;
  • git add - adds new or changed files in your local working directory;
  • git commit - is used to save changes to the local repository;
  • git log - is used to view the history of committed changes within a Git repository;
  • git checkout - is used to switch between branches in a repository;
  • git branch - is used to create a new branch;
  • git push - is used to push the local repository content to a remote repository;
  • git pull - is used to fetch and merge code changes from the remote repository to the local repository;
  • git merge - allows to merge branches from Git (the pull request's commits are squashed into a single commit);
  • git clone - used to point to an existing repo and make a clone or copy of that repo at in a new directory, at another location.

.gitignore file is a text file that tells Git which files to ignore when committing your project to the GitHub repository / . gitignore is located in the root directory of your repo.

Advanced commands

  • git revert - used to record some new commits to reverse the effect of some earlier commits (possibly faulty ones);
  • git reset - used to move the repository back to a previous commit, discarding any changes made after that commit;
  • git rebase - allows rewriting commits from one Git branch to another branch;
  • git cherry-pick - allows to choose a commit from one branch and apply it to another;
  • git stash - saves the uncommitted changes locally, allowing you to make changes, switch branches, perform other Git operations, and then come back and re-apply them later on.
